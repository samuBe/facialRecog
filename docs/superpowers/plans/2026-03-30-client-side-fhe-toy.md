# Client-Side FHE Toy PoC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Validate the full round-trip: browser encrypts (openfhe-wasm) → server computes homomorphic addition (openfhe-python) → browser decrypts result, proving client-side FHE is feasible before integrating into the facial recognition app.

**Architecture:** The browser uses openfhe-wasm to generate CKKS keys and encrypt values. It sends serialized keys and ciphertexts (base64 JSON) to the server. The server uses openfhe-python to deserialize, perform EvalAdd, serialize the result, and return it. The browser decrypts locally. Both sides use identical hardcoded CKKS parameters.

**Tech Stack:** OpenFHE C++ (built from source), openfhe-python (macOS ARM64 build), openfhe-wasm (Emscripten build), FastAPI, vanilla JS

**Key Discovery:** heir_py's pybind11 Ciphertext objects have NO serialization methods. The `openfhe` pip package only ships Linux x86_64 wheels. Therefore, the toy endpoint uses openfhe-python built from source (NOT heir_py) for all server-side FHE operations. The existing heir_py endpoints are untouched.

    cc.Enable(openfhe.PKESchemeFeature.KEYSWITCH)

def test_python_serialize_roundtrip():
    """Verify openfhe-python can serialize and deserialize a ciphertext."""
    cc = make_ckks_context()
    keys = cc.KeyGen()

    ptxt = cc.MakeCKKSPackedPlaintext([1.0, 2.0, 3.0])
    ct = cc.Encrypt(keys.publicKey, ptxt)

    # Serialize ciphertext
    ct_str = cc.SerializeCiphertext(ct)  # or SerializeToString — check API
    assert len(ct_str) > 0, "Serialized ciphertext should not be empty"

    # Deserialize
    ct2 = cc.DeserializeCiphertext(ct_str)

    # Decrypt and verify
    result = cc.Decrypt(ct2, keys.secretKey)
    result.SetLength(3)
    assert abs(result[0].real - 1.0) < 1e-3
    assert abs(result[1].real - 2.0) < 1e-3
    assert abs(result[2].real - 3.0) < 1e-3

```python
# server/fhe_toy_routes.py
"""Toy FHE endpoints for validating client-side encryption round-trip."""

from __future__ import annotations

import base64
import logging
from typing import Any

import openfhe
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel

logger = logging.getLogger("fhe.toy")

router = APIRouter(prefix="/fhe", tags=["fhe-toy"])

# In-memory store for client keys (single-client toy)
_client_state: dict[str, Any] = {}

# Hardcoded CKKS parameters — must match client-side openfhe-wasm config
CKKS_PARAMS = {
    "mult_depth": 1,
    "scaling_mod_size": 50,
    "first_mod_size": 60,
    "ring_dim": 8192,
    "batch_size": 8192,
}


class KeyUploadRequest(BaseModel):
    crypto_context: str  # base64-encoded serialized CryptoContext
    public_key: str  # base64-encoded serialized PublicKey
    eval_keys: str | None = None  # base64-encoded eval keys (needed for EvalMult, optional for EvalAdd)


class ToyAddRequest(BaseModel):
    ct_a: str  # base64-encoded serialized Ciphertext
    ct_b: str  # base64-encoded serialized Ciphertext


@router.post("/keys")
def upload_keys(payload: KeyUploadRequest) -> dict[str, str]:
    """Receive and cache client-generated CKKS keys.

    Note: eval_keys are optional for the toy (EvalAdd doesn't need them).
    Phase B (dot product) will require eval keys for EvalMult.
    """
    try:
        cc_bytes = base64.b64decode(payload.crypto_context)
        pk_bytes = base64.b64decode(payload.public_key)

        # Deserialize using openfhe
        # NOTE: Exact deserialization API depends on openfhe-python version
        # This may use DeserializeCryptoContext, DeserializeFromString, etc.
        cc = openfhe.DeserializeCryptoContext(cc_bytes)
        pk = openfhe.DeserializePublicKey(pk_bytes, cc)

        _client_state["cc"] = cc
        _client_state["pk"] = pk

        if payload.eval_keys:
            ek_bytes = base64.b64decode(payload.eval_keys)
            # Deserialize and load eval keys into context
            # cc.DeserializeEvalMultKey(ek_bytes)  # or similar
            _client_state["has_eval_keys"] = True

        logger.info("[keys] Client keys cached successfully")
        return {"status": "ok"}

    except Exception as exc:
        logger.exception("[keys] Failed to deserialize client keys")
        raise HTTPException(status_code=400, detail=str(exc)) from exc


@router.post("/toy-add")
def toy_add(payload: ToyAddRequest) -> dict[str, str]:
    """Perform homomorphic addition on two client-encrypted ciphertexts."""
    if "cc" not in _client_state:
        raise HTTPException(status_code=400, detail="No client keys uploaded. POST /fhe/keys first.")

    try:
        cc = _client_state["cc"]

        ct_a_bytes = base64.b64decode(payload.ct_a)
        ct_b_bytes = base64.b64decode(payload.ct_b)

        # Deserialize ciphertexts
        ct_a = openfhe.DeserializeCiphertext(ct_a_bytes, cc)
        ct_b = openfhe.DeserializeCiphertext(ct_b_bytes, cc)

        # Homomorphic addition — server never sees plaintext
        ct_sum = cc.EvalAdd(ct_a, ct_b)

        # Serialize result
        result_bytes = openfhe.SerializeCiphertext(ct_sum)
        result_b64 = base64.b64encode(result_bytes).decode()

        logger.info("[toy-add] Homomorphic addition complete, result size: %d bytes", len(result_bytes))
        return {"result": result_b64}

    except Exception as exc:
        logger.exception("[toy-add] Failed")
        raise HTTPException(status_code=500, detail=str(exc)) from exc
```
